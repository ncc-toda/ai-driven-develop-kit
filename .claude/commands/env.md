# 環境整備ワークフロー

開発環境、CI/CD、依存関係などの環境整備を行うワークフローです。

## 使い方

````bash
/wf-env 整備内容
```bash

---

## STEP 1: 作業ログ作成

`docs/tasks/ai-logs/YYYY-MM-DD_{slug}.md` に作業ログを作成すること。

## STEP 2: 影響範囲の確認

以下を確認すること:

- 変更が影響する範囲（開発環境、CI/CD、依存関係など）
- 既存の動作に影響がないか
- チームメンバーへの影響

## STEP 3: 環境整備の実施

変更を実施すること。
問題が発生した場合は作業ログに記録すること。

## STEP 4: 検証

以下を確認すること:

- 変更が正しく適用されているか
- 既存の機能が壊れていないか

### 静的解析・テスト

```bash
devbox run lint
devbox run format
devbox run test:ts  # または test:dart
```bash

## STEP 5: ドキュメント更新

環境整備に伴い、以下のドキュメント更新を検討すること:

- README.md
- CLAUDE.md
- その他関連ドキュメント

## STEP 6: 完了

1. 作業ログの「振り返り」セクションを記入すること
2. 必要に応じてドキュメント・ルールを更新すること
3. Git コミットすること
4. ユーザーに完了報告すること
````
