# プロジェクト初期化ワークフロー

テンプレートをコピーした直後に実行し、自分のプロジェクトとして初期化するワークフローです。

## 使い方

```bash
/init-project プロジェクトの概要（名前、目的、主な機能など）
```

---

## STEP 1: 現状確認

以下を確認すること:

- 現在のディレクトリがテンプレートからコピーされたものか
- `.git` ディレクトリの状態（既存のテンプレート履歴が残っているか）
- `devbox` がインストールされているか

## STEP 2: プロジェクト情報のヒアリング

ユーザーに以下を確認すること:

- **プロジェクト名**: リポジトリ名・表示名
- **概要**: 1-2 文でのプロジェクト説明
- **目的**: 何を達成したいか
- **主な機能**: 想定している機能（概要レベル）
- **対象ユーザー**: 誰が使うか
- **技術スタック（任意）**: 決まっていれば

## STEP 3: Git リポジトリの初期化

以下を実施すること:

1. 既存の `.git` ディレクトリを削除
2. 新規 Git リポジトリとして初期化
3. デフォルトブランチを `main` に設定

```bash
rm -rf .git
git init
git branch -M main
```

## STEP 4: README.md の更新

`README.md` をプロジェクトに合わせて更新すること:

- プロジェクト名
- 概要説明
- テンプレートへの参照を削除
- 必要に応じてセクションを調整

## STEP 5: CLAUDE.md の更新

`CLAUDE.md` をプロジェクトに合わせて更新すること:

- 「プロジェクト概要」セクションをプロジェクト固有の内容に変更
- 現在のフェーズを確認・設定
- 必要に応じてクイックリファレンスを調整

## STEP 6: テンプレートファイルの整理

以下を実施すること:

1. `TEMPLATE_USAGE.md` を削除
2. `docs/tasks/ai-logs/` 配下のサンプルログを削除（存在する場合）
3. `docs/tasks/pre-release/` の内容を確認し、不要なら整理

## STEP 7: 開発環境の確認

以下を実施すること:

1. `devbox shell` で開発環境に入る
2. `devbox run setup` を実行
3. Lint/Format が動作することを確認

```bash
devbox shell
devbox run setup
devbox run lint
devbox run format
```

## STEP 8: 初期コミット

以下を実施すること:

1. 全ファイルをステージング
2. 初期コミットを作成

```bash
git add -A
git commit -m "Initial commit from ai-driven-develop-kit"
```

## STEP 9: 完了

ユーザーに以下を報告すること:

- 初期化が完了したこと
- 次のステップとして `/requirements` の実行を推奨
- 作成・更新されたファイルの一覧

**次のステップ:**

```bash
/requirements プロジェクトの詳細な要件
```
