# ミニマリストレビュー

最小限の原則の観点でコードをレビューします。
過剰な実装や不要な複雑さを検出します。

## 使い方

````bash
/review-minimalist [対象ファイルやコード]
```bash

---

## 確認観点

- 余計な変更を含んでいないか
  - 依頼された範囲外の変更
  - 「ついでに」の修正
  - 関係のないリファクタリング
- 不要な抽象化をしていないか
  - 1箇所でしか使わないインターフェース
  - 過度なデザインパターンの適用
  - 将来の拡張を見越した過剰設計
- 依頼された範囲を超えていないか
  - スコープクリープ
  - 未承認の機能追加
- YAGNI（You Aren't Gonna Need It）原則に反していないか
  - 使われない機能
  - 必要になるかもしれない機能の先行実装
  - 過度な汎用化
- 過度な最適化をしていないか
  - 計測なしの最適化
  - 可読性を犠牲にした最適化
  - マイクロ最適化
- シンプルな解決策を選んでいるか
  - より単純な代替案はないか
  - 複雑さに見合う価値があるか

## 出力

過剰な部分があれば、具体的な削減案を提示すること。
「なぜその複雑さが必要なのか」を問うこと。
````
