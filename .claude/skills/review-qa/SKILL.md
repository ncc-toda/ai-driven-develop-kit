# QA レビュー

テスト・品質保証の観点でコードをレビューします。

## 使い方

````bash
/review-qa [対象ファイルやコード]
```bash

---

## 指示

Task ツールで `qa` subagent を使用してレビューを実行してください。

## 確認観点

- テストケースは十分か
  - 正常系: 期待通りの入力で期待通りの出力
  - 異常系: 不正な入力でのエラーハンドリング
  - 境界値: 最小値、最大値、空、null
- テストが実際に意図した動作を検証しているか
  - アサーションは適切か
  - テスト名は検証内容を表しているか
- リグレッションテストが必要な箇所はカバーされているか
  - バグ修正には対応するテストがあるか
  - 重要な機能は十分にテストされているか
- テストの可読性・保守性は十分か
  - Arrange-Act-Assert パターン
  - テストデータの管理
  - 重複の排除
- モック・スタブの使い方は適切か
  - 外部依存の適切な分離
  - モックの過剰使用を避ける
- E2E テストが必要な機能か
  - ユーザーフロー全体のテスト
  - 統合テストとの使い分け

## 出力

問題があれば具体的な改善案を提示すること。
テストケースの追加が必要な場合は、具体的なテストケースを提案すること。
````
