# セキュリティレビュー

セキュリティの観点でコードをレビューします。

## 使い方

````bash
/review-security [対象ファイルやコード]
```bash

---

## 指示

Task ツールで `security` subagent を使用してレビューを実行してください。

## 確認観点

- 認証・認可の漏れがないか
- バリデーション不足がないか
- 機密情報の露出がないか
- OWASP Top 10 に該当する脆弱性がないか
  - インジェクション（SQL, NoSQL, OS コマンド, LDAP）
  - 認証の不備
  - 機密データの露出
  - XML 外部エンティティ（XXE）
  - アクセス制御の不備
  - セキュリティ設定ミス
  - クロスサイトスクリプティング（XSS）
  - 安全でないデシリアライゼーション
  - 既知の脆弱性を持つコンポーネントの使用
  - 不十分なロギングとモニタリング
- SQLインジェクション、XSS、CSRF などの対策がされているか
- 依存パッケージに既知の脆弱性がないか
- シークレット（API キー、パスワード等）がハードコードされていないか
- HTTPS/TLS が適切に使用されているか
- 入力値のサニタイズが行われているか

問題があれば具体的な改善案を提示すること。
````
