# モバイルレビュー

モバイルアプリ共通の観点でコードをレビューします。

## 使い方

````bash
/review-mobile [対象ファイルやコード]
```bash

---

## 指示

Task ツールで `mobile` subagent を使用してレビューを実行してください。

## 確認観点

- オフライン対応を考慮しているか
  - ローカルキャッシュ
  - オフライン時の UI フィードバック
  - データ同期戦略
  - Optimistic UI
- バッテリー消費を考慮しているか
  - バックグラウンド処理の最小化
  - 位置情報の効率的な使用
  - ネットワークリクエストのバッチ処理
  - アニメーションの最適化
- ネットワーク状態の変化に対応しているか
  - 接続状態の監視
  - リトライ機構
  - タイムアウト処理
  - 低速回線への対応
- プッシュ通知の扱いは適切か
  - 権限リクエストのタイミング
  - 通知の種類と優先度
  - バッジ・サウンドの設定
  - ディープリンクとの連携
- ディープリンクの扱いは適切か
  - Universal Links / App Links
  - URL スキーム
  - パラメータの検証
  - 未認証時の処理
- 各プラットフォームのガイドラインに準拠しているか
  - iOS: Human Interface Guidelines
  - Android: Material Design
  - プラットフォーム固有の UI パターン
- アプリのライフサイクルを考慮しているか
  - フォアグラウンド/バックグラウンド遷移
  - 状態の保存と復元
  - メモリ警告への対応

問題があれば具体的な改善案を提示すること。
````
