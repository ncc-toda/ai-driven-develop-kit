# 認知とインタラクション（原則 4-8）

## 4. メンタルモデル (Mental Model) [P1]

**定義**: ユーザーが想像する利用モデルに合った構成と動きにする

### メンタルモデルの形成方法

1. **既存の概念活用**: フォルダ、ゴミ箱、デスクトップ
2. **操作イディオム**: ダブルクリック=実行、右クリック=メニュー、スワイプ=ページング
3. **物理世界の法則**: スクロールの慣性、上から下への視線移動
4. **文化的概念**: 赤=停止/危険、緑=進行/安全

**良い例**: Apple iBooks（本棚メタファで直感的な整理）、ゴミ箱（削除 ≠ 完全消去、回復可能）

**悪い例**: 技術用語の使用、不適切なメタファ（フォルダアイコンなのにクリックすると検索）

## 5. シグニファイア (Signifier) [P1]

**定義**: 操作対象が見えていて、その意味が一目でわかるようにする

### シグニファイアの種類

1. **形状的**: 立体感のあるボタン、ホバー効果
2. **テキスト的**: 明確な動詞ラベル（「送信する」「保存」）
3. **位置的**: 主要アクション=右下、閉じる=右上
4. **状態的**: Default、Hover、Active、Focus、Disabled

### 実装ポイント

```css
/* ボタンの例 */
.button {
  cursor: pointer;
  transition: all 0.2s ease;
}
.button:hover {
  transform: translateY(-1px);
}
.button:disabled {
  background: #ccc;
  cursor: not-allowed;
}
```

**良い例**: Material Design のボタン（影付き立体感、一貫したホバー効果）、GitHub のボタン階層（Primary/Secondary/Danger）

**悪い例**: フラットデザインの極端化、アイコンのみのナビゲーション（ラベルなし）

## 6. マッピング (Mapping) [P1]

**定義**: 操作する所と結果が反映される所の対応を把握できるようにする

### 良いマッピングの原則

1. **空間的対応**: コントロールの配置 ≡ 対象の配置
2. **形状的対応**: 大きなスライダー=大きな変化量
3. **色による対応**: 緑=ポジティブ、赤=ネガティブ
4. **時間的対応**: 即座のフィードバック、リアルタイム表示

**良い例**: Photoshop のレイヤーパネル（レイヤーの上下順序=画像の重ね順）、エアコンパネル

**悪い例**: コントロールと対象が離れている、設定パネルとプレビューが別画面

---

## 7. 直接操作 (Direct Manipulation) [P1]

**定義**: 画面上のオブジェクトに直接触れて操作している感覚を与える

### 直接操作の要素

1. **可視性**: 操作対象が見えている
2. **即座のフィードバック**: リアルタイム更新
3. **可逆性**: Undo/Redo、ESC でキャンセル
4. **段階的学習**: 単純操作 → 複合操作

### 実装例

```javascript
// ドラッグ操作
element.addEventListener("dragstart", (e) => {
  e.target.classList.add("dragging");
});

// スライダーのリアルタイム反映
slider.addEventListener("input", (e) => {
  targetElement.style.opacity = e.target.value / 100;
});
```

**良い例**: Apple Photos の写真整理（ドラッグでアルバム間移動）、Sketch の図形操作

**悪い例**: 設定ダイアログ方式、フィードバックなし（ドラッグ中にオブジェクトが消える）

## 8. 視覚ゲシュタルト (Visual Gestalt) [P1]

**定義**: 人の知覚特性を利用して要素同士のグループ関係を効果的に示す

### ゲシュタルトの法則

1. **近接**: 近いものは関連していると認識
2. **類似**: 似たものは同じグループと認識
3. **閉鎖**: 不完全な形も完結した形として認識
4. **連続**: 連続したパターンを認識
5. **共通運命**: 同じ動きをするものはグループ
6. **図と地**: 前景と背景を区別

### ゲシュタルト実装ポイント

```css
/* 余白による近接 */
.section {
  margin-bottom: var(--space-xl);
}
.related-items {
  gap: var(--space-sm);
}

/* カードによる閉鎖 */
.card {
  border: 1px solid #e0e0e0;
  border-radius: 8px;
  padding: 16px;
}

/* モーダルによる図と地 */
.modal-overlay {
  background: rgba(0, 0, 0, 0.5);
}
.modal-content {
  background: white;
  z-index: 1001;
}
```

**良い例**: Apple 設定画面（関連設定項目が視覚的にグループ化）、GitHub Issue 表示

**悪い例**: すべてに罫線を引く、ホワイトスペース不足（情報が詰まりすぎて読みにくい）
