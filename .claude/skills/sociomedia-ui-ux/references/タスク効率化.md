# タスク効率化（原則 18-22）

## 18. 複雑性保存の法則 (Law of Conservation of Complexity) [P2]

**定義**: 複雑性は減らせず移動できるのみ。ユーザー側からシステム側へ移動する

### 複雑性の移動先

1. **システム側（バックエンド）**: 自動化、AI 分類
2. **デザイナー側**: プログレッシブディスクロージャー、スマートデフォルト
3. **エンジニア側**: バックグラウンド処理、キャッシュ、予測ロード
4. **ユーザー側**: 最小化すべき

### 実装例

```javascript
// ❌ ユーザーに複雑性を押し付け
takePicture(settings) {
  return this.camera.capture({
    aperture: settings.aperture,    // F 値の理解が必要
    shutterSpeed: settings.shutterSpeed,  // 露出計算が必要
    iso: settings.iso,              // ノイズ耐性の知識が必要
  });
}

// ✅ システムが複雑性を負担
takePicture(scene = 'auto') {
  const optimalSettings = this.calculateOptimalSettings(scene);
  return this.camera.capture(optimalSettings);  // ユーザーは撮影するだけ
}
```

### プログレッシブディスクロージャー

```html
<!-- レベル 1: 基本調整（全ユーザー向け）-->
<div class="basic-adjustments">
  <label>明るさ: <input type="range" /></label>
</div>

<!-- レベル 2: 詳細調整（中級者向け）-->
<details>
  <summary>詳細調整</summary>
  <label>ハイライト: <input type="range" /></label>
</details>
```

## 19. タスクコヒーレンス (Task Coherence) [P1]

**定義**: 過去の行動パターンを活用してエクスペリエンスを最適化する

### 活用方法

1. **最近使った項目**: ファイル、検索語、連絡先
2. **履歴とブックマーク**: 頻度分析、パターン認識
3. **前回の設定保存**: フォーム状態、フィルター条件
4. **セッション復元**: タブ、スクロール位置
5. **パーソナライゼーション**: スキルレベル、使用パターン

### コヒーレンス実装例

```javascript
class RecentFileManager {
  recordFileAccess(filePath, context) {
    const record = {
      filePath,
      timestamp: Date.now(),
      context: {
        project: context.project,
        timeOfDay: new Date().getHours(),
      },
    };
    this.recentFiles.unshift(record);
    this.saveRecentFiles();
  }

  getRelevantRecentFiles(currentContext) {
    return this.recentFiles
      .filter((record) => this.isRelevant(record, currentContext))
      .sort((a, b) => this.calculateRelevance(b) - this.calculateRelevance(a));
  }
}
```

### プライバシー配慮

- 履歴の削除機能
- データ保存期間設定
- プライベートモード対応

## 20. メジャーなタスクに最適化 (Optimize for Major Tasks) [P1]

**定義**: 80% のユーザーが使う 20% の機能を優先し、多数派のタスクに最適化

### メジャータスクの特定方法

1. ユーザー行動データ分析
2. ヒートマップ分析
3. ユーザーインタビュー
4. サポート問い合わせ分析

### メジャータスク実装例

```html
<!-- Tier 1: 最重要（80% が毎日使用）-->
<section class="primary-actions">
  <button class="btn-primary-large">新規作成</button>
  <button class="btn-primary-large">検索</button>
</section>

<!-- Tier 2: 重要（50% が週 1 回使用）-->
<section class="secondary-actions">
  <button class="btn-secondary">インポート</button>
  <button class="btn-secondary">設定</button>
</section>

<!-- Tier 3: 補助（20% が月 1 回使用）-->
<details class="advanced-features">
  <summary>高度な機能</summary>
  <button class="btn-tertiary">一括編集</button>
</details>
```

## 21. ショートカットを用意する (Provide Shortcuts) [P2]

**定義**: 熟練ユーザー向けの効率的なパスを提供する

### ショートカットの種類

1. **キーボードショートカット**: Ctrl+S、Cmd+Z
2. **ジェスチャー**: スワイプ、ピンチ
3. **クイックアクション**: 右クリックメニュー、長押し
4. **最近使った項目**: ファイル、コマンド
5. **テンプレート/マクロ**: 繰り返し操作の記録

### 実装ポイント

```javascript
// キーボードショートカット
document.addEventListener("keydown", (e) => {
  if (e.ctrlKey || e.metaKey) {
    switch (e.key) {
      case "s":
        e.preventDefault();
        save();
        break;
      case "z":
        e.preventDefault();
        undo();
        break;
      case "f":
        e.preventDefault();
        openSearch();
        break;
    }
  }
});
```

### 表示方法

- メニュー項目にショートカットキー表示
- ツールチップで案内
- 設定画面でカスタマイズ可能

## 22. パースエージョン (Persuasion) [P2]

**定義**: 行動科学を活用して望ましい行動を促進する

### パースエージョンの技法

1. **ソーシャルプルーフ**: 「○ 人が購入しました」
2. **スカーシティ**: 「残り 3 個」「本日限り」
3. **レシプロシティ**: 無料お試し、特典
4. **コミットメント**: 小さな承諾から大きな承諾へ
5. **ゲーミフィケーション**: ポイント、バッジ、ランキング

### パースエージョン実装例

```html
<!-- ソーシャルプルーフ -->
<div class="social-proof">
  <span>⭐ 4.8 (2,345 件のレビュー)</span>
  <span>🔥 今日 127 人が購入</span>
</div>

<!-- スカーシティ -->
<div class="scarcity-indicator">
  <span class="urgent">残り 3 個</span>
  <span class="timer">セール終了まで: 2:34:56</span>
</div>

<!-- ゲーミフィケーション -->
<div class="streak-indicator">
  <span>🔥 5 日連続ログイン！</span>
  <span>あと 2 日でゴールドバッジ獲得</span>
</div>
```

### 倫理的配慮

- ダークパターンを避ける
- 透明性を保つ
- ユーザーの利益を優先
- 誤解を招く表現を避ける
