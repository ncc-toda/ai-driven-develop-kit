# チェックリストと活用ガイド

## 原則の優先度

| 優先度      | 原則                                                             | 説明                     |
| ----------- | ---------------------------------------------------------------- | ------------------------ |
| **P0 必須** | 1 シンプル, 2 簡単, 3 一貫性, 9 主導権, 13 エラー回避            | 使いやすい UI の最低要件 |
| **P1 推奨** | 4-8 認知系, 10-12 ユーザー系, 14-15 制約系, 19-20 効率系, 23OOUI | 優れた UX 実現に重要     |
| **P2 理想** | 16-17 法則系, 18 複雑性, 21-22 効率系, 24 ビュー                 | 洗練された UX の高度手法 |

## フェーズ別チェックリスト

### 1. 企画・要件定義

**適用原則**: オブジェクトベース, メジャータスク最適化

- [ ] ターゲットユーザーを特定したか
- [ ] ユーザーの主要タスクを 80/20 で優先順位付けしたか
- [ ] ユーザーの関心の対象（オブジェクト）を特定したか
- [ ] オブジェクト同士の関係性を整理したか

### 2. 情報設計

**適用原則**: シンプル, 視覚ゲシュタルト, 一貫性, ヒックの法則

- [ ] 情報の優先順位は明確か
- [ ] 関連する情報はグループ化されているか
- [ ] 選択肢の数は 7±2 以内か
- [ ] ナビゲーション階層は 3 階層以下か

### 3. ワイヤーフレーム

**適用原則**: シグニファイア, マッピング, フィッツの法則, 直接操作

- [ ] 押せる要素は押せそうに見えるか
- [ ] 重要なボタンは大きく、近い場所に配置したか
- [ ] タッチターゲットは最小 44×44px か
- [ ] 主要な操作は 1-2 クリックで完了するか

### 4. ビジュアルデザイン

**適用原則**: 一貫性, 視覚ゲシュタルト, シグニファイア

- [ ] カラーパレットは体系的に定義されているか
- [ ] タイポグラフィの階層は明確か
- [ ] ボタンスタイルは一貫しているか
- [ ] 同じ機能は同じ見た目になっているか

### 5. 実装・開発

**適用原則**: 直接操作, モードレス, エラー回避, プリコンピュテーション

- [ ] デザイン通りの見た目が実現されているか
- [ ] モーダルダイアログは最小限か
- [ ] フォームバリデーションはリアルタイムか
- [ ] 適切なデフォルト値が設定されているか

### 6. テスト・評価

**適用原則**: すべての原則を検証

- [ ] 主要タスクは問題なく完了できるか
- [ ] 初回利用者でも使い方を理解できるか
- [ ] エラー発生時に回復できるか
- [ ] アクセシビリティガイドラインに準拠しているか

## コンポーネント別チェック

### ナビゲーション

- [ ] 主要セクションは 7 項目以内か
- [ ] 現在位置が明確に表示されているか
- [ ] ロゴクリックでトップに戻れるか

### ボタン・ CTA

- [ ] プライマリボタンは 1 画面 1 つか
- [ ] 最小 44×44px を確保しているか
- [ ] ホバー/フォーカス状態が定義されているか
- [ ] 無効化状態が適切に表現されているか

### フォーム

- [ ] 必須項目が明確に区別されているか
- [ ] プレースホルダーだけでなくラベルがあるか
- [ ] エラーメッセージは具体的で建設的か
- [ ] 自動フォーマット（電話番号等）が実装されているか

### モーダル

- [ ] モーダルを使う必要性があるか
- [ ] 閉じる方法が複数あるか（×、ESC、背景クリック）
- [ ] フォーカス管理が適切か

## よくある問題と対処法

| 問題               | 関連原則                                     | 対処法                       |
| ------------------ | -------------------------------------------- | ---------------------------- |
| 機能が多すぎて複雑 | シンプル, メジャータスク, ヒック             | 80/20 で機能厳選、段階的開示 |
| ユーザーが迷う     | シグニファイア, マッピング, 記憶             | ボタンを明確に、パンくず設置 |
| エラーが多発       | エラー回避, コンストレイント, 主導権         | 制約で防止、Undo 提供        |
| 操作が面倒         | 簡単, プリコンピュテーション, ショートカット | ステップ削減、デフォルト値   |
| 学習コストが高い   | メンタルモデル, 一貫性, 複雑性保存           | 既存概念活用、パターン統一   |

## プロジェクトタイプ別ガイド

### Web アプリケーション

重要原則: シンプル, 一貫性, 主導権, オブジェクトベース, エラー回避

- [ ] ブラウザの戻るボタンに対応
- [ ] URL が意味を持つ構造
- [ ] ショートカットキー活用
- [ ] ページロード 3 秒以内

### モバイルアプリ

重要原則: フィッツの法則, シンプル, 直接操作, メジャータスク

- [ ] 親指ゾーンを考慮した配置
- [ ] タッチターゲット 44×44px 以上
- [ ] 片手操作で主要機能が使える
- [ ] iOS HIG / Material Design 準拠

### 管理画面・ダッシュボード

重要原則: オブジェクトベース, 視覚ゲシュタルト, メジャータスク, ショートカット

- [ ] 重要指標がひと目でわかる
- [ ] フィルタリング/ソート機能充実
- [ ] 一括操作機能
- [ ] エクスポート機能

### EC サイト

重要原則: メジャータスク, プリコンピュテーション, パースエージョン, 簡単

- [ ] 商品画像は高画質で複数角度
- [ ] 決済は 3 ステップ以内
- [ ] カートに入れるボタンが目立つ
- [ ] ゲストチェックアウト可能

## デザインレビューチェック

### P0 原則（必須）

- [ ] **シンプル**: 不要な要素は削除されているか
- [ ] **一貫性**: デザインパターンは統一されているか
- [ ] **エラー回避**: エラーが起きにくい設計か
- [ ] **主導権**: ユーザーがコントロールできるか
- [ ] **シグニファイア**: 操作方法が明確にわかるか

### 技術品質

- [ ] カラーコントラストは WCAG AA 以上か
- [ ] フォントサイズは最小 16px か
- [ ] ページロード 3 秒以内か
- [ ] キーボードナビゲーション可能か
