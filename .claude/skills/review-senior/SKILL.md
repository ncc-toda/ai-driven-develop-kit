# シニアエンジニアレビュー

品質・設計の観点でコードをレビューします。

## 使い方

````bash
/review-senior [対象ファイルやコード]
```bash

---

## 指示

Task ツールで `reviewer` subagent を使用してレビューを実行してください。

## 確認観点

- エッジケースを考慮しているか
  - 空配列、null、undefined
  - 境界値（0、負数、最大値）
  - 同時実行、競合状態
- 命名は適切か
  - 意図が明確に伝わる名前
  - プロジェクトの命名規則に準拠
  - 略語の適切な使用
- パフォーマンスに問題がないか
  - 不要なループ、再計算
  - N+1 問題
  - メモリリーク
- エラーハンドリングは適切か
  - 例外の適切なキャッチと処理
  - エラーメッセージの明確さ
  - リカバリー処理
- コードの可読性・保守性は十分か
  - 適切なコメント
  - 関数・クラスの責務の明確さ
  - 複雑度の管理
- 適切な抽象化がされているか
  - 過度な抽象化を避ける
  - 必要な抽象化は行う
  - SOLID 原則の適用
- テストカバレッジは十分か

## 出力

問題があれば具体的な改善案を提示すること。
````
