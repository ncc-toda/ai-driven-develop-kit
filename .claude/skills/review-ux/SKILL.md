# UI/UX レビュー

ユーザー体験の観点でコードをレビューします。

## 使い方

````bash
/review-ux [対象ファイルやコード]
```bash

---

## 指示

Task ツールで `frontend` subagent を使用してレビューを実行してください。

## 確認観点

- 既存の意匠との一貫性があるか
  - デザインシステムへの準拠
  - 色、フォント、スペーシングの統一
  - コンポーネントの再利用
- インタラクションは自然か
  - クリック/タップ領域の適切さ
  - フィードバックの即時性
  - アニメーションの適切さ
- アクセシビリティを考慮しているか
  - スクリーンリーダー対応
  - キーボード操作
  - 色のコントラスト
  - フォーカス管理
- レスポンシブ対応は適切か
  - ブレークポイントの設定
  - モバイルファースト
  - タッチ操作の考慮
- ローディング状態やエラー状態の表示は適切か
  - スケルトンスクリーン
  - プログレスインジケータ
  - エラーメッセージの明確さ
  - リトライ機能
- ユーザーにとって分かりやすい UI になっているか
  - 直感的な操作
  - 適切なラベル
  - ヘルプ・ツールチップ

## 出力

問題があれば具体的な改善案を提示すること。
````
