# 作業ログ

作業の記録と学びを蓄積するための手順です。

## 基本ルール

- **保存先**: `docs/tasks/ai-logs/YYYY-MM-DD_{slug}.md`
- **命名**: 日付と作業内容を表す slug（例: `2026-01-28_login-validation-fix.md`）
- **更新タイミング**: 各ステップの完了時、問題に遭遇した時に随時更新すること

## テンプレート

```markdown
# {タイトル}

## 概要

{作業内容の説明}

## 作業内容

{実施した内容を記録}

## 検証結果

{テストや動作確認の結果}

---

## 発生した問題・はまったこと

### 問題1: {問題の概要}

**現象**:

**原因**:

**解決策**:

**今後の注意点**:

## 振り返り

### 学んだこと

-

### ドキュメント・ルール更新の提案

| 対象ファイル | 更新内容 |
| ------------ | -------- |
|              |          |
```

## 作業完了時の振り返り

作業完了時に以下を確認すること:

1. 「発生した問題・はまったこと」セクションが記録されているか
2. 「今後の注意点」が明確に書かれているか
3. ドキュメント・ルール更新が必要な場合、提案を記載したか
4. 提案がある場合、実際に更新を行ったか
