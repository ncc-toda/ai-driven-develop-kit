# プロジェクトセットアップワークフロー

このワークフローに従ってプロジェクトをゼロから構築します。
ユーザーから受け取った作業内容を元に、以下の手順を順番に実行してください。

---

## STEP 1: 作業ログ作成

[作業ログ](../_parts/log.md) に従い、作業ログを作成すること。

## STEP 2: 設計確認

初期設計書が存在する場合は内容を確認すること。
存在しない場合は [初期設計ワークフロー](./initial-design.md) を先に実施すること。

## STEP 3: リポジトリ初期化

以下を実施すること:

- Git リポジトリの初期化
- .gitignore の設定
- README.md の作成（プロジェクト概要）

## STEP 4: 開発環境構築

設計に基づき以下を構築すること:

- パッケージマネージャの設定（devbox, npm, pub 等）
- 依存関係のインストール
- 環境変数の設定（.env.example 等）

## STEP 5: コード品質ツール設定

以下を設定すること:

- リンター（ESLint, dart analyze 等）
- フォーマッター（Prettier, dart format 等）
- 型チェック（TypeScript, Dart 等）
- pre-commit フック（必要に応じて）

## STEP 6: ディレクトリ構造作成

設計に基づきディレクトリ構造を作成すること:

- 各ディレクトリの作成
- 必要な初期ファイルの配置
- README や .gitkeep の配置

## STEP 7: 動作確認

以下を確認すること:

- ビルドが通ること
- リンター・フォーマッターが動作すること
- 基本的なコマンドが実行できること

## STEP 8: ドキュメント整備

以下を作成・更新すること:

- README.md（セットアップ手順）
- CLAUDE.md（AI 向けガイド、必要に応じて）
- その他必要なドキュメント

## STEP 9: 完了

1. [作業ログ](../_parts/log.md) の「作業完了時の振り返り」を実施すること
2. [Git 運用ルール](../../git/git-index.md) に従いコミットすること
3. ユーザーに完了報告すること
