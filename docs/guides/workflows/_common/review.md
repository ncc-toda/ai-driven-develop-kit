# セルフレビュー手順

この手順は他のワークフローから呼び出される共通手順です。
以下の役割の観点で変更内容をレビューし、問題があれば修正すること。

## レビュー役割

### 共通

| 役割                                           | 観点             |
| ---------------------------------------------- | ---------------- |
| [セキュリティエンジニア](./roles/security.md)  | セキュリティ     |
| [シニアエンジニア](./roles/senior-engineer.md) | 品質・設計       |
| [QA エンジニア](./roles/qa.md)                 | テスト・品質保証 |
| [UI/UX デザイナー](./roles/ui-ux-designer.md)  | ユーザー体験     |
| [ミニマリスト](./roles/minimalist.md)          | 最小限の原則     |

### 言語・FW 別

| 役割                                              | 観点         |
| ------------------------------------------------- | ------------ |
| [シニア TypeScript](./roles/senior-typescript.md) | TypeScript   |
| [シニア Dart/Flutter](./roles/senior-dart.md)     | Dart/Flutter |
| [シニア Swift/iOS](./roles/senior-swift.md)       | Swift/iOS    |

### プラットフォーム別

| 役割                                               | 観点               |
| -------------------------------------------------- | ------------------ |
| [シニア Web フロントエンド](./roles/senior-web.md) | Web フロントエンド |
| [シニアモバイル](./roles/senior-mobile.md)         | モバイル共通       |

## レビュー手順

1. 作業内容に応じて適切な役割を選択すること
2. 各役割のファイルを参照し、その観点でレビューを行うこと
3. 問題があれば修正すること
4. レビュー結果を作業ログに記録すること

## 注意事項

- すべての役割でレビューする必要はない（作業内容に応じて選択）
- 言語・FW 別、プラットフォーム別は該当するもののみ
- UI/UX デザイナーは UI 変更がある場合のみ
