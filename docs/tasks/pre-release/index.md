## 運用ルール

- タスクには連番 ID を振る（001, 002, ...）
- 詳細が必要なタスクは `detail/` に個別ファイルを作成
- ファイル名: `{ID}-{簡潔な説明}.md`（例: `001-setup-devbox.md`）
- リリース後、残りのタスクは GitHub Issue へ移行する
