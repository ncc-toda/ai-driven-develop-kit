## 運用ルール

- シンプルなチェックリストで管理
- `/requirements` 完了時 → 設計タスクを追加
- `/design-doc` 完了時 → 実装タスクを追加
- リリース後、残りのタスクは GitHub Issue へ移行

### 記載例

```markdown
## 📋 TODO

### 設計フェーズ

- [ ] 技術スタック決定
- [ ] 環境セットアップ
- [ ] API 設計

### 実装フェーズ

- [ ] ユーザー認証
- [ ] ログイン画面
```
